# Tasks

Define background tasks using the `@pb.task()` decorator. Tasks receive a payload and can be triggered via fan-out, programmatically, or from the dashboard.

## Defining a Task

```python
@pb.task("send-email", retries=2, timeout="15s")
def send_email(ctx, payload):
    deliver_email(payload["id"], payload["recipient"])
    ctx.log("Sent email", id=payload["id"])
    return {"sent": payload["id"]}
```

## Typed Payloads

Task handlers can accept a typed second parameter for autocomplete and validation. Works with dataclasses and Pydantic models:

```python
from dataclasses import dataclass

@dataclass
class EmailPayload:
    to: str
    subject: str
    priority: int = 1

@pb.task("send-email", retries=3)
def send_email(ctx, payload: EmailPayload):
    send_mail(payload.to, payload.subject)
    ctx.log("Sent", to=payload.to)
```

With Pydantic:

```python
from pydantic import BaseModel

class OrderPayload(BaseModel):
    order_id: str
    amount: float
    email: str

@pb.task("process-order")
def process_order(ctx, payload: OrderPayload):
    ctx.log("Processing", order_id=payload.order_id)
```

All three styles are supported:

| Style | Signature | Payload access |
|-------|-----------|----------------|
| No param | `def job(ctx)` | `ctx.payload["key"]` |
| Raw dict | `def job(ctx, payload)` | `payload["key"]` |
| Typed | `def job(ctx, payload: MyType)` | `payload.key` |
