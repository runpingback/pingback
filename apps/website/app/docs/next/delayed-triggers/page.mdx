# Delayed Triggers

Trigger a task with a delay — the execution will be scheduled to run after the specified time.

## Usage

```typescript
import { PingbackClient } from "@usepingback/next";

const pingback = new PingbackClient({
  apiKey: process.env.PINGBACK_API_KEY!,
});

// Delay with a human-readable string
await pingback.trigger("send-email", { to: "user@example.com" }, { delay: "15m" });

// Delay in seconds
await pingback.trigger("send-email", { to: "user@example.com" }, { delay: 900 });
```

## Delay Formats

| Format | Example | Description |
|--------|---------|-------------|
| Seconds (integer) | `900` | Delay by 900 seconds |
| Seconds | `"30s"` | Delay by 30 seconds |
| Minutes | `"15m"` | Delay by 15 minutes |
| Hours | `"2h"` | Delay by 2 hours |
| Days | `"1d"` | Delay by 1 day |
| Combined | `"1d2h30m"` | Delay by 1 day, 2 hours, 30 minutes |

Maximum delay: 30 days.

## Response

The response includes the scheduled execution time:

```typescript
const { executionId, scheduledAt } = await pingback.trigger(
  "send-reminder",
  { userId: "123" },
  { delay: "2h" }
);

console.log(scheduledAt); // "2026-04-27T15:00:00.000Z"
```

## Use Cases

- **Reminder emails** — send a follow-up 24 hours after signup
- **Cooldown periods** — wait before retrying an external API call
- **Scheduled notifications** — notify users at a specific time relative to an event
- **Deferred processing** — batch work to run during off-peak hours
